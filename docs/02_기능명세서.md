## 3. 기능별 상세

### F-001 카페 목록 조회
- **사용자 동작:** 메인 페이지 진입 또는 필터 조건 변경 시 자동 호출
- **호출 API:** `GET /api/spaces/`
- **성공 시 UI:** 지도 마커 갱신 + 목록 카드 갱신
- **빈 상태:** "조건에 맞는 카페가 없습니다" 안내
- **에러 상태:** "카페 정보를 불러오지 못했어요. 새로고침 해주세요."

### F-002 카페 필터링
- **사용자 동작:** 필터 패널에서 최소 점수 선택 → 자동 재조회
- **쿼리 파라미터:** `min_score_plug`, `min_score_wifi`, `min_score_noise`, `min_score_comfort`, `min_score_table`
- **호출 API:** `GET /api/spaces/?min_score_plug=4&min_score_wifi=4`

### F-003 카페 정렬
- **사용자 동작:** 정렬 드롭다운 선택
- **쿼리 파라미터:** `ordering` (예: `-score_comfort`, `-score_plug`)
- **기본값:** `-score_comfort` (눈치 점수 높은 순)

### F-004 카페 단일 조회
- **사용자 동작:** 마커 클릭 또는 목록 카드 클릭
- **호출 API:** `GET /api/spaces/<id>/`
- **응답:** id, name, address, latitude, longitude, 5대 점수, total_review_count

### F-005 카카오맵 지도 표시
- **사용 SDK:** 카카오맵 JavaScript SDK
- **인증:** Kakao JavaScript Key (카카오 개발자 콘솔 발급)
- **마커 색상:** 종합 점수 구간별 구분 (초록 ≥ 4.0 / 노랑 ≥ 3.0 / 빨강 < 3.0)

### F-006 마커 클릭 팝업
- **표시 항목:** 카페명, 주소, 콘센트·와이파이·소음·눈치·테이블 점수
- **UI:** 카카오맵 CustomOverlay 또는 InfoWindow

### F-007 필터 패널 UI
- **항목:** 각 지표별 최소 점수 (1~5, 기본값 1)
- **UX:** 변경 즉시 API 재호출 (디바운스 300ms 적용 권장)

## 4. 화면 흐름도

```
[메인 /] ──▶ 카카오맵 로드 ──▶ GET /api/spaces/ ──▶ 마커 렌더링
   │
   ├── 필터 변경 ──▶ GET /api/spaces/?min_score_plug=4&... ──▶ 마커 갱신
   │
   ├── 마커 클릭 ──▶ GET /api/spaces/<id>/ ──▶ 팝업 표시
   │
   └── 목록 뷰 [/list] ──▶ 카드 리스트 ──▶ 카드 클릭 ──▶ [/cafe/:id] 상세
```
